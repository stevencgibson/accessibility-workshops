Interface elements should be operable

2.1.1 Keyboard

  • don't add keyboard shortcuts - they'll override screenreader shortcuts
  • pretty much everything must be keyboard-accessible

Exceptions:

  • drawing tools
  • games

2.1.2 No keyboard trap

Users should be able to access any feature (e.g. datepicker, modal, tabs...) and move away from it using the keyboard.

2.2.1 Timing adjustable

  • avoid putting time constraints on content
  • allow users to adjust time constraints up to 10x
  • allow users to reduce the frequency of auto-updating content by up to 10x

2.2.2 Pause, stop, hide

  • avoid autoplaying scrollable content like banners
  • at least provide a means of pausing
  • probably applies to auto-updating search results
  • allow users to set a preference on autoplaying media

Some users may prefer media to autoplay while some may not.

Navigating (by keyboard) to media to play it is just as complex as navigating to media to pause it.

Allowing users to set a preference on autoplaying media gives users more control.

2.4.1 Bypass blocks

Make it easy for keyboard users to skip repeated content.

  // Can be hidden by default.
  // Must be visible on focus.
  // Must set focus on the target upon click.

  <a href="#content">Skip to content</a>

  <nav>
    <ul>
      <li><a href="/">Link 1</a></li>
      ...
    </ul>
  </nav>

  <main id="content" tabindex="-1">
  ...
  </main>

2.4.2 Page title

  • always set a page's title
  • helps screenreader users figure out where they are
  • also helps SEO
  • update the title if the content changes (e.g. AJAX search)

2.4.3 Focus order

Focus order should make sense to keyboard users.

Keyboard-focusable elements

  // The default for focusable elements
  <input type="text" tabindex="0" />

  // Altering focus order
  <input type="text" tabindex="1" />

  // Removing from keyboard, but not programmatic, focus order
  <input type="text" tabindex="-1" />

2.4.3 Focus order

Non-keyboard-focusable elements

  // The default for non-focusable elements
  <span tabindex="-1"></span>
  
  // Making the non-focusable focusable
  <span tabindex="0"></span>

  // Altering focus order
  <span tabindex="1"></span>

Try to avoid using tabindex by coding content in the order in which it will be displayed.

2.4.4 Link purpose

Screenreader users can access a list of all links so ensure that the purpose of all links is clear.

Avoid

  <a href="/">Click Here</a>

A note on the title attribute

  <a href="/products" title="Products">Products</a>

It's f****** useless.

  • poor browser support
  • no touch support
  • only announced by screenreaders on some elements

Is handy for keyboard users if used as an alternative to labels for form fields.
